🚀 git init

Crea un nuevo repositorio de Git en tu carpeta actual.

📌 ¿Qué hace?

El comando git init convierte una carpeta cualquiera en un repositorio Git, es decir, un proyecto donde Git controlará los cambios y versiones.

⚡ Sintaxis básica

git init

📝 Ejemplo práctico

Imagina que empiezas un nuevo proyecto en una carpeta llamada mi-app. Lo primero es inicializar Git para comenzar a llevar el control de versiones. 

mkdir mi-app
cd mi-app
git init

🔎 Desglose del comando

📌 git → ejecuta Git, el sistema de control de versiones.

📌 init → inicializa el repositorio en la carpeta actual, creando la carpeta oculta .git con toda la configuración interna.

💡 Tips & Trucos

⚡ No uses git init dentro de un repositorio ya existente: causará conflictos.

⚡ Usa git status justo después de un git init para confirmar que todo está vacío.

⚡ Si clonas un repositorio con git clone, no necesitas git init.

✔️ Buenas prácticas

✔️ Inicializa el repositorio en la raíz de tu proyecto, no en subcarpetas.

✔️ Después de un git init, lo primero es configurar tu identidad:

git config user.name "Tu Nombre"
git config user.email "tu@email.com"

⚠️ Errores comunes

    ❌ Ejecutar git init en cada carpeta → solo debe hacerse en la raíz del proyecto.

    ❌ Confundir git init con git clone: init crea un repo vacío, clone copia uno ya existente.
