🚀 git init

Crea un nuevo repositorio de Git en tu carpeta actual.

📌 ¿Qué hace?

El comando git init convierte una carpeta cualquiera en un repositorio Git, es decir, un proyecto donde Git controlará los cambios.

⚡ Sintaxis básica

git init

🏢 Ejemplo real

Imagina que empiezas un nuevo proyecto en una carpeta llamada mi-app. Lo primero es inicializar Git para comenzar a llevar el control de versiones. 

mkdir mi-app
cd mi-app
git init
🔎 Desglose del comando
  • git

    Es el programa que ejecutamos: Git, el sistema de control de versiones.

  • init

    Orden que inicializa el repositorio en la carpeta actual, creando la carpeta oculta .git con toda la configuración interna.

💡 Tips & Trucos

No uses git init dentro de un repositorio ya existente: causará conflictos.

Usa git status justo después de un git init para confirmar que todo está vacío.

Si clonas un repositorio con git clone, no necesitas git init.

⚠️ Errores comunes

  • ❌ Ejecutar git init en cada carpeta → no es necesario, solo en la raíz del proyecto.
  • ❌ Confundir git init con git clone. Init crea un repo vacío, clone copia uno ya existente.

✔️ Buenas prácticas

Inicializa el repositorio en la raíz de tu proyecto, no en subcarpetas.

Después de un git init, lo primero es configurar tu identidad: git config user.name "Tu Nombre" git config user.email "tu@email.com"
