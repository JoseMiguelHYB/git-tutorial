🔎 git grep

Busca cadenas de texto en el historial y archivos versionados de Git.

📌 ¿Qué hace?

El comando git grep permite buscar patrones de texto dentro del contenido de los archivos que están en el repositorio, incluyendo versiones específicas, ramas o commits.

⚡ Sintaxis básica

git grep <patrón> [rango]

👉 Por defecto busca el patrón en todos los archivos rastreados por Git.

📝 Ejemplo práctico

Si quieres buscar la palabra function en el proyecto:

git grep "function"

Salida típica:

src/app.js:12:function initApp() {
src/utils.js:45:export function calculate() {

👉 También puedes buscar en un commit específico:

git grep "TODO" abc1234

⚙️ Opciones comunes

⚡ git grep "texto" → busca el texto en todos los archivos del repositorio.

⚡ git grep -n "texto" → muestra también el número de línea.

⚡ git grep -i "texto" → búsqueda sin distinguir mayúsculas/minúsculas.

⚡ git grep "texto" branch_name → busca solo en una rama específica.

⚡ git grep "regex" → admite expresiones regulares.

✔️ Buenas prácticas

✔️ Usa -n para ubicar rápidamente dónde aparece el texto.

✔️ Combínalo con --and o --or para búsquedas más complejas.

✔️ Úsalo en lugar de grep normal cuando quieras asegurarte de buscar solo en archivos rastreados por Git.

⚠️ Errores comunes

⚠️ Creer que busca en archivos no rastreados → solo revisa lo que Git ya sigue.

⚠️ Olvidar que diferencia mayúsculas/minúsculas si no se usa -i.
