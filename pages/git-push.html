🚀 git push

Sube tus cambios confirmados al repositorio remoto.

📌 ¿Qué hace?

El comando git push envía tus commits locales a un repositorio remoto. Se utiliza después de hacer git add y git commit para compartir tu trabajo con otros.

⚡ Sintaxis básica

git push <remoto> <rama>

📝 Ejemplo práctico

# Subir cambios de la rama actual al remoto "origin"
git push origin main

# Subir todas las ramas
git push --all origin

# Subir etiquetas (tags)
git push --tags

👉 Normalmente se usa origin como remoto y main o master como rama principal.

⚙️ Opciones comunes

⚡ git push → sube la rama actual al remoto predeterminado.

⚡ git push origin main → sube la rama main al remoto origin.

⚡ git push --all → sube todas las ramas.

⚡ git push --tags → sube todas las etiquetas.

⚡ git push -u origin main → establece main como rama de seguimiento.

✔️ Buenas prácticas

✔️ Siempre haz git pull antes de un git push para evitar conflictos.

✔️ Usa -u la primera vez para vincular tu rama local con el remoto.

✔️ Sube ramas con nombres claros (ej: feature/login, bugfix/navbar).

⚠️ Errores comunes

⚠️ "rejected non-fast-forward" → ocurre cuando el remoto tiene cambios que no tienes localmente.

⚠️ Olvidar usar -u y tener que escribir siempre git push origin main.

⚠️ Subir directamente a main sin pasar por ramas de desarrollo en proyectos colaborativos.
