🌍 git remote

Gestiona repositorios remotos en tu proyecto Git.

📌 ¿Qué hace?

El comando git remote permite administrar las conexiones con repositorios remotos. Con él puedes listar, agregar, renombrar o eliminar remotos asociados a tu repositorio local.

⚡ Sintaxis básica

git remote [opciones]

📝 Ejemplo práctico

# Listar remotos
git remote -v

# Agregar un remoto
git remote add origin https://github.com/usuario/proyecto.git

# Renombrar un remoto
git remote rename origin upstream

# Eliminar un remoto
git remote remove origin

👉 Normalmente se usa origin como nombre del remoto principal.

⚙️ Opciones comunes

⚡ git remote → lista los remotos configurados.

⚡ git remote -v → lista remotos con sus URLs.

⚡ git remote add <nombre> <url> → agrega un nuevo remoto.

⚡ git remote rename <viejo> <nuevo> → renombra un remoto.

⚡ git remote remove <nombre> → elimina un remoto.

✔️ Buenas prácticas

✔️ Usa origin como nombre del remoto principal.

✔️ Si trabajas con forks, usa nombres claros como upstream para el remoto original.

✔️ Revisa con git remote -v antes de hacer push/pull para evitar errores.

⚠️ Errores comunes

⚠️ Usar una URL incorrecta al agregar el remoto → causará fallos en git push y git pull.

⚠️ Confundir origin con upstream en proyectos colaborativos.

⚠️ Olvidar eliminar remotos que ya no se usan, lo que puede causar confusión.
