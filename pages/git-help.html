❓ git help

Consulta la ayuda integrada de Git: comandos, guías y documentación.

📌 ¿Qué hace?

El comando git help muestra información sobre Git y sus comandos disponibles, o abre páginas del manual en distintos formatos según las opciones usadas.

⚡ Sintaxis básica

git help [opciones] [comando/guía]

⚙️ Opciones comunes

-a / --all → Muestra todos los comandos disponibles.

-g / --guides → Muestra las guías conceptuales de Git.

-c / --config → Lista todas las variables de configuración disponibles.

-i / --info → Abre la ayuda en formato info.

-m / --man → Abre la página de manual.

-w / --web → Abre la ayuda en formato HTML en tu navegador.

📝 Ejemplo práctico

# Muestra comandos comunes
git help

# Muestra todos los comandos disponibles
git help -a

# Listar guías conceptuales
git help --guides

# Ver configuración disponible
git help --config

# Abrir manual en formato man
git help -m git

# Abrir ayuda en HTML en navegador
git help -w commit

✔️ Buenas prácticas

✔️ Usa git help -a para explorar comandos desconocidos.

✔️ Recurre a git help <comando> antes de intentar un comando potente.

✔️ Prefiere el formato --web si quieres una experiencia visual navegable.

⚠️ Errores comunes

⚠️ Usar --all sin saber que incluye comandos externos y alias puede abrumar.

⚠️ Algunos nombres como user-manual pueden fallar al abrir, según cómo se interprete el prefijo git-.
