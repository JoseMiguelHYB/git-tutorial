🗑️ git rm

Elimina archivos del área de trabajo y del índice de Git.

📌 ¿Qué hace?

El comando git rm elimina archivos del área de trabajo y también del índice (staging area), marcándolos para ser eliminados en el próximo commit.

⚡ Sintaxis básica

git rm <archivo>

👉 También se puede usar con múltiples archivos o patrones.

📝 Ejemplo práctico

Si deseas eliminar un archivo llamado old.js del proyecto:

git rm old.js
git commit -m "Eliminado old.js del proyecto"

Si quieres eliminar varios archivos de una carpeta:

git rm logs/*.log
git commit -m "Limpieza de archivos de log"

⚙️ Opciones comunes

⚡ git rm archivo → elimina el archivo del directorio y del repositorio.

⚡ git rm -r carpeta/ → elimina una carpeta completa.

⚡ git rm --cached archivo → elimina el archivo solo del repositorio, pero lo mantiene en tu directorio local.

⚡ git rm -f archivo → fuerza la eliminación si el archivo tiene cambios sin guardar.

✔️ Buenas prácticas

✔️ Usa --cached para dejar de rastrear archivos que deben permanecer en tu máquina (ej. configuraciones locales).

✔️ Revisa con git status antes de hacer commit para confirmar qué archivos serán eliminados.

✔️ No uses -f a menos que estés seguro de querer eliminar cambios sin guardar.

⚠️ Errores comunes

⚠️ Creer que git rm borra archivos solo del repositorio → también los elimina de tu directorio de trabajo (a menos que uses --cached).

⚠️ Olvidar hacer commit después de git rm → el archivo seguirá apareciendo como eliminado pendiente.
