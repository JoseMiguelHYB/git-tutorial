🔄 git pull

Descarga y fusiona cambios del repositorio remoto en tu repositorio local.

📌 ¿Qué hace?

El comando git pull se usa para traer (fetch) los cambios de un repositorio remoto y fusionarlos (merge o rebase) automáticamente en tu rama local actual.

⚡ Sintaxis básica

git pull <remoto> <rama>

📝 Ejemplo práctico

# Descargar y fusionar cambios de la rama main desde origin
git pull origin main

# Descargar y fusionar de la rama actual (si ya está configurada con -u)
git pull

👉 Por defecto, git pull trae los cambios del remoto configurado con git push -u.

⚙️ Opciones comunes

⚡ git pull → descarga y fusiona cambios de la rama remota configurada.

⚡ git pull origin main → descarga cambios de main desde origin.

⚡ git pull --rebase → reescribe tus commits encima de los nuevos cambios (historial más limpio).

⚡ git fetch + git merge → equivalente a un git pull manual.

✔️ Buenas prácticas

✔️ Haz git pull con frecuencia para mantener tu rama actualizada.

✔️ Usa git pull --rebase en ramas de características (features) para un historial más ordenado.

✔️ Si hay conflictos, resuélvelos antes de hacer un git push.

⚠️ Errores comunes

⚠️ Conflictos al fusionar → ocurre cuando los mismos archivos fueron editados en local y remoto.

⚠️ No usar git pull antes de git push → puede causar rechazos de push.

⚠️ Confusión entre git fetch y git pull (fetch solo descarga, pull descarga y fusiona).
