💾 git commit

Guarda los cambios preparados en el historial del repositorio.

📌 ¿Qué hace?

El comando git commit guarda los cambios que están en el staging area dentro del historial del repositorio. Cada commit crea un “punto de control” en el proyecto, permitiendo volver atrás o revisar la evolución.

⚡ Sintaxis básica

git commit -m "Mensaje descriptivo"

👉 El flag -m permite escribir el mensaje directamente.

📝 Ejemplo práctico

# Guardamos un fix con mensaje técnico y claro
git commit -m "Fix bug in user authentication"

🔎 Desglose del comando

📌 git → ejecuta Git, el sistema de control de versiones.

📌 commit → guarda los cambios preparados (staged).

📌 -m → agrega un mensaje en línea.

📌 "Fix bug in user authentication" → mensaje conciso y claro.

⚙️ Opciones comunes y avanzadas

⚡ git commit -a -m "msg" → añade y commitea archivos rastreados sin git add.

⚡ git commit --amend -m "msg" → modifica el último commit.

⚡ git commit --no-edit --amend → corrige el último commit sin cambiar el mensaje.

⚡ git commit --signoff → añade la firma del autor (DCO).

⚡ git commit --allow-empty -m "Marcador" → crea un commit vacío útil como marcador.

⚡ git commit --no-verify → omite los hooks de validación.

⚡ git commit --date="2023-12-01" → establece una fecha personalizada.

✔️ Buenas prácticas

    ✔️ Usa el imperativo: Add, Fix, Update.

    ✔️ Mensajes cortos y claros (máx. 50 caracteres en la primera línea).

    ✔️ Si necesitas más detalle, añade una segunda línea.

    ✔️ Agrupa cambios relacionados en un commit pequeño y enfocado.

⚠️ Errores comunes

    ⚠️ No uses mensajes genéricos como "Update" o "Fix".

    ⚠️ Evita commits gigantes con muchos cambios mezclados.
