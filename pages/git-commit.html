💾 git commit

Guarda los cambios preparados en el historial del repositorio.

📌 ¿Qué hace?

El comando git commit guarda los cambios preparados (staged) en el historial del repositorio.

⚡ Sintaxis básica

git commit -m "Mensaje"

🏢 Ejemplo real

Imagina que tu jefe te dice: “Miguel, necesitamos que hagas commit con la nueva función de login para que el equipo pueda probarla.” 

# Guardamos el cambio con un mensaje claro
git commit -m "Add login function with validations"
🔎 Desglose del comando
  • git
    Es el programa que ejecutamos. Git es el sistema de control de versiones que gestiona el historial del proyecto.
  • commit
    Acción que guarda los cambios preparados (staged) en el historial, creando un nuevo “punto de control”.
  • -m
    Opción para incluir el mensaje directamente en la línea de comandos. Sin -m, se abriría el editor de texto por defecto.
  • "Add login function with validations"
    El mensaje del commit. Debe ser breve y claro para que otros (y tú mismo en el futuro) entiendan qué se cambió.

💡 Tips & Trucos

    Puedes usar git commit -am "msg" para añadir y commitear en un solo paso.

    Escribe mensajes cortos pero descriptivos (menos de 50 caracteres si puedes).

    Usa git commit --amend para corregir el último commit.

⚠️ Errores comunes

  • ❌ Olvidar -m → Git abre el editor por defecto.
  • ❌ Usar mensajes genéricos como "Update" o "Fix".
  • ✅ Solución: mensajes claros como "Fix login bug en formulario".

✔️ Buenas prácticas

    Usa el imperativo: Add, Fix, Update en vez de Added o Fixed.

    Agrupa cambios relacionados en un solo commit.

    Evita commits gigantes, hazlos pequeños y enfocados.
