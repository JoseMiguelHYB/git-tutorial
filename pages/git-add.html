➕ git add

Agrega cambios al área de preparación (staging area).

📌 ¿Qué hace?

El comando git add toma los cambios de tus archivos (nuevos o modificados) y los coloca en el staging area, preparando esos cambios para ser confirmados en el próximo git commit.

⚡ Sintaxis básica

git add [opciones] <ruta-de-archivo>

👉 Puedes agregar un archivo específico, varios archivos o todos los cambios del directorio.

📝 Ejemplo práctico

Agregar un archivo específico:

git add index.html

Agregar todos los cambios en el proyecto:

git add .

🔎 Desglose del comando

📌 index.html → Solo agrega ese archivo al staging area.

📌 . → Agrega todos los archivos modificados en el directorio actual.

⚙️ Opciones comunes y avanzadas

⚡ git add archivo1 archivo2 → agrega varios archivos a la vez.

⚡ git add *.html → agrega todos los archivos que cumplan con el patrón.

⚡ git add -p → permite añadir cambios de forma interactiva (por fragmentos).

⚡ git add -A → agrega todos los cambios, incluyendo eliminaciones.

✔️ Buenas prácticas

✔️ Usa git status para revisar qué archivos están en staging antes de hacer commit.

✔️ Agrega solo los archivos necesarios, evita usar git add . sin revisar.

✔️ Utiliza git add -p para confirmar solo los cambios que quieres.

⚠️ Errores comunes

⚠️ Creer que git add hace un commit → solo prepara los cambios, no los confirma.

⚠️ Usar git add . sin revisar → puede añadir archivos no deseados.

⚠️ Olvidar añadir cambios → el commit saldrá vacío o incompleto.
