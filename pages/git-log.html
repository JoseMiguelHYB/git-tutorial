📜 git log

Explora el historial de commits de tu proyecto.

📌 ¿Qué hace?

El comando git log muestra el historial de commits del repositorio. Puedes ver quién hizo cada cambio, cuándo se hizo y el mensaje del commit.

⚡ Sintaxis básica

git log [opciones]

👉 Sin opciones, muestra todos los commits en orden cronológico inverso.

📝 Ejemplo práctico

git log

Salida típica:

commit 9c1a0d5d1a2e3b4f9a7d5c0a3e92d3e91a2f4e1b
Author: Juan Pérez <juanperez@example.com>
Date:   Thu Aug 29 10:32:15 2024 +0200

    Añadida la funcionalidad de login

commit 8a3f0b5c2b1e7d3f5a1b2c3d4e5f6a7b8c9d0e1f
Author: Ana López <analopez@example.com>
Date:   Wed Aug 28 18:15:42 2024 +0200

    Corregido error en validación de formularios

⚙️ Opciones comunes

⚡ git log --oneline → muestra los commits en una sola línea.

⚡ git log --graph --oneline --decorate → historial en forma de árbol.

⚡ git log -p → muestra los cambios completos (diffs) de cada commit.

⚡ git log --author="Juan" → filtra commits por autor.

⚡ git log --since="2 weeks ago" → muestra commits recientes.

✔️ Buenas prácticas

✔️ Usa --oneline para revisar rápido el historial.

✔️ Combina --graph para entender ramas y merges.

✔️ Filtra por autor o fecha para auditorías y debugging.

⚠️ Errores comunes

⚠️ Pensar que git log muestra los cambios de archivos → eso lo hace git diff o git show.

⚠️ Confundir --oneline con un historial completo → es solo un resumen.
