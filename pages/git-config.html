⚙️ git config

Configura opciones de usuario y ajustes globales de Git.

📌 ¿Qué hace?

El comando git config permite establecer variables de configuración que determinan el comportamiento de Git. Por ejemplo: tu nombre de usuario, tu correo, el editor de texto o reglas de formato.

⚡ Sintaxis básica

git config [opciones] nombre.valor "contenido"

👉 La opción --global aplica el cambio para todos los repositorios.

📝 Ejemplo práctico

Lo primero que debes hacer tras instalar Git es configurar tu identidad:

git config --global user.name "Juan Pérez"
git config --global user.email "juan@example.com"

🔎 Desglose del comando

📌 --global → Aplica la configuración a nivel de usuario (todos los repositorios).

📌 user.name → Define el nombre de usuario que aparecerá en los commits.

📌 user.email → Define el correo asociado a los commits.

⚙️ Opciones comunes y avanzadas

⚡ git config --list → muestra todas las configuraciones activas.

⚡ git config --system → aplica la configuración a todo el sistema.

⚡ git config --local → configura opciones solo para el repositorio actual.

⚡ git config core.editor "code --wait" → establece un editor por defecto.

⚡ git config --global color.ui auto → activa colores en la salida de Git.

✔️ Buenas prácticas

✔️ Configura tu identidad (user.name y user.email) justo después de instalar Git.

✔️ Usa correos profesionales o asociados a tu cuenta de GitHub/GitLab.

✔️ Configura también un editor preferido (ej: VS Code, Vim, Nano).

⚠️ Errores comunes

⚠️ No configurar user.name y user.email antes de hacer commits.

⚠️ Usar un correo distinto al asociado en GitHub/GitLab → los commits pueden no vincularse.

⚠️ Creer que git config cambia configuraciones del sistema operativo → solo afecta a Git.
