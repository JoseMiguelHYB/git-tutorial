📂 git mv

Mueve o renombra archivos en tu repositorio con seguimiento de historial.

📌 ¿Qué hace?

El comando git mv se usa para mover o renombrar archivos dentro de un repositorio. Git registra el cambio como una operación de rename o move, conservando el historial del archivo.

⚡ Sintaxis básica

git mv <archivo_origen> <archivo_destino>

👉 También puedes usarlo para mover archivos a carpetas nuevas.

📝 Ejemplo práctico

Si quieres renombrar main.js a app.js:

git mv main.js app.js
git commit -m "Renombrado main.js a app.js"

Si quieres mover un archivo a otra carpeta:

git mv index.html src/index.html
git commit -m "Movido index.html a carpeta src"

⚙️ Opciones comunes

⚡ git mv archivo nuevo_nombre → renombra un archivo.

⚡ git mv archivo carpeta/ → mueve un archivo a una carpeta.

⚡ git mv -f archivo nuevo_nombre → fuerza el movimiento si ya existe el destino.

✔️ Buenas prácticas

✔️ Usa git mv en lugar de comandos del sistema operativo para mantener el historial de Git.

✔️ Confirma los cambios con git status antes de hacer commit.

✔️ Evita sobreescribir archivos a menos que realmente lo necesites (usa -f con cuidado).

⚠️ Errores comunes

⚠️ Mover un archivo con el explorador en lugar de git mv → Git lo verá como borrado y nuevo archivo.

⚠️ Olvidar hacer commit después del movimiento → los cambios quedarán sin registrar.
