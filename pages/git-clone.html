📥 git clone

Clona un repositorio existente en una nueva carpeta local.

📌 ¿Qué hace?

El comando git clone crea una copia completa de un repositorio remoto (o local) en tu máquina. Descarga todos los commits, ramas y archivos, configurando automáticamente la conexión con el remoto.

⚡ Sintaxis básica

git clone [opciones] <url-repositorio> [nombre-directorio]

👉 Si no especificas [nombre-directorio], Git usará el nombre del repositorio.

📝 Ejemplo práctico

Clonar un repositorio desde GitHub:

git clone https://github.com/usuario/proyecto.git

🔎 Desglose del comando

📌 https://github.com/usuario/proyecto.git → URL del repositorio remoto.

📌 proyecto → Carpeta que se creará en tu máquina con todos los archivos y commits.

⚙️ Opciones comunes y avanzadas

⚡ git clone <url> nuevo-nombre → clona el repo en una carpeta con nombre personalizado.

⚡ git clone --branch develop <url> → clona directamente la rama develop.

⚡ git clone --depth 1 <url> → clona solo el último commit (shallow clone).

⚡ git clone --recurse-submodules <url> → clona también los submódulos.

✔️ Buenas prácticas

✔️ Usa --depth 1 si solo necesitas la última versión y no todo el historial.

✔️ Verifica la URL (HTTPS o SSH) antes de clonar para evitar problemas de permisos.

✔️ Cambia el nombre de la carpeta al clonar si trabajas con múltiples copias del mismo repositorio.

⚠️ Errores comunes

⚠️ Usar una URL incorrecta → error de acceso al repositorio.

⚠️ No tener permisos (SSH o credenciales) → fallo en la autenticación.

⚠️ Confundir git clone con git pull → el primero crea una copia nueva, el segundo actualiza una existente.
