📊 git status

Muestra el estado actual del repositorio y sus archivos.

📌 ¿Qué hace?

El comando git status muestra el estado de los archivos en tu repositorio: cuáles han sido modificados, cuáles están en el área de staging, cuáles no están siendo rastreados y si tu rama está adelantada o atrasada respecto al remoto.

⚡ Sintaxis básica

git status [opciones]

👉 Sin opciones, muestra el estado completo de los archivos y la rama actual.

📝 Ejemplo práctico

Después de modificar algunos archivos, puedes ejecutar:

git status

Resultado típico:

On branch main
Your branch is up to date with 'origin/main'.

Changes not staged for commit:
  modified:   index.html

Untracked files:
  script.js

no changes added to commit

🔎 Desglose del comando

📌 Changes not staged → archivos modificados pero no agregados con git add.

📌 Untracked files → archivos que aún no están en el control de versiones.

📌 Mensajes sobre branch → te dicen si tu rama está sincronizada con el remoto.

⚙️ Opciones comunes y avanzadas

⚡ git status -s → salida corta (simplificada).

⚡ git status -b → muestra también información de la rama.

⚡ git status --ignored → lista también los archivos ignorados.

✔️ Buenas prácticas

✔️ Ejecuta git status frecuentemente para revisar qué archivos están listos para commit.

✔️ Usa la opción -s cuando quieras un resumen rápido.

✔️ Siempre revisa el estado antes de hacer un commit importante.

⚠️ Errores comunes

⚠️ Creer que git status hace cambios → solo muestra información, no modifica nada.

⚠️ Ignorar los archivos no rastreados → pueden ser importantes y olvidarse de añadirlos.
