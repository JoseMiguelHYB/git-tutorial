🔍 git diff

Compara los cambios en los archivos de tu repositorio.

📌 ¿Qué hace?

El comando git diff muestra las diferencias entre distintas versiones de los archivos: los cambios pendientes respecto al área de staging, los commits, ramas o incluso respecto al repositorio remoto.

⚡ Sintaxis básica

git diff [opciones] [rango]

👉 Sin parámetros, compara archivos modificados con el último commit.

📝 Ejemplo práctico

Si modificamos index.html pero no lo agregamos aún al staging:

git diff

Salida típica:

diff --git a/index.html b/index.html
--- a/index.html
+++ b/index.html
@@ -1,5 +1,6 @@
 <html>
   <head>
-    <title>Mi página</title>
+    <title>Mi página actualizada</title>
   </head>
   <body>
+    <p>Nuevo párrafo</p>
   </body>
 </html>

🔎 Desglose del comando

📌 Líneas con - → fueron eliminadas.

📌 Líneas con + → fueron agregadas.

📌 Los bloques @@ indican la ubicación de los cambios en el archivo.

⚙️ Opciones comunes

⚡ git diff → muestra cambios en el directorio de trabajo (no staged).

⚡ git diff --staged → muestra cambios entre staging y el último commit.

⚡ git diff HEAD → compara todo (staging + no staged) con el último commit.

⚡ git diff main..feature → compara dos ramas.

⚡ git diff commit1 commit2 → compara dos commits específicos.

✔️ Buenas prácticas

✔️ Revisa siempre git diff antes de hacer commit.

✔️ Usa --staged para confirmar que lo que vas a commitear es correcto.

✔️ Para revisiones más visuales, combina con GUIs como gitk o extensiones de VS Code.

⚠️ Errores comunes

⚠️ Creer que git diff modifica archivos → solo muestra diferencias.

⚠️ Olvidar usar --staged y pensar que se están viendo los cambios listos para commit.
